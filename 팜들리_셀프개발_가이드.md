# 팜들리 셀프 개발 가이드 (비전문가용)

약사 두 분이 직접 앱을 만들기 위한 안내서입니다. 전문 용어는 비유로 풀어서 설명합니다.

---

## 0. 용어부터 쉽게

| 용어 | 쉽게 말하면 |
|---|---|
| 프론트엔드 | 손님이 보는 **매대와 인테리어** (화면) |
| 백엔드/서버 | 손님 눈에 안 보이는 **조제실** (데이터 처리) |
| 데이터베이스(DB) | **약장 서랍** — 약국·상품·주문 정보를 정리해 보관하는 곳 |
| Next.js | 화면을 만드는 도구 세트 (매대 진열 도구) |
| Supabase | 조제실+약장을 통째로 빌려주는 서비스 (서버를 직접 짓지 않아도 됨) |
| Vercel | 만든 앱을 인터넷에 올려주는 서비스 (개업) |
| Cursor / Claude Code | 말로 시키면 코드를 대신 짜주는 AI 비서 |
| localhost:3000 | 내 컴퓨터 안에서만 보이는 **가오픈 매장** 주소 |

---

## 1. 준비물 설치 (첫날, 1~2시간)

1. **Node.js** 설치 — nodejs.org 접속 → LTS 버전 다운로드 → 설치. (앱을 돌리는 엔진입니다)
2. **Cursor** 설치 — cursor.com 접속 → 다운로드. (AI가 들어있는 코드 편집기. 화면은 메모장 같지만 옆에 채팅창이 있어요)
3. **GitHub** 가입 — github.com. (코드를 백업하는 클라우드. 나중에 배포할 때도 필요)

## 2. 앱 첫 실행 (10분)

1. Cursor를 열고 → File → Open Folder → 이 폴더(`pharmdly-app`)를 엽니다.
2. 위 메뉴에서 Terminal → New Terminal 을 누르면 아래에 검은 입력창이 생깁니다.
3. 입력창에 아래를 한 줄씩 입력하고 엔터:

```
npm install
npm run dev
```

4. 브라우저에서 `http://localhost:3000` 접속 → 환자용 화면이 나오면 성공!
5. 새 탭에서 `http://localhost:3000/pharmacy` 를 열면 약국용 화면입니다.
   환자 화면에서 주문을 보내면 약국 화면에 바로 떠요. (지금은 같은 컴퓨터 안에서만)

> 막히면? 에러 메시지를 그대로 복사해서 Cursor 채팅창(또는 저에게) 붙여넣고 "이거 해결해줘"라고 하세요. 이게 셀프 개발의 핵심 기술입니다.

## 3. 지금 앱의 상태 (정직하게)

지금 만들어드린 앱은 **뼈대**입니다. 프로토타입의 핵심 흐름(약국 선택 → 담기 → 주문 → 약사 승인 → 준비완료 → 픽업)이 실제로 작동하지만:

- 주문이 **내 컴퓨터 브라우저 안에만** 저장됩니다. (다른 사람 폰과 연결 안 됨 → 4단계에서 해결)
- 동네 인증, 지도, 트렌드, 알림, 상품 추가 요청은 아직 없습니다. (프로토타입 참고해서 하나씩 추가)
- 회원가입/로그인이 없습니다.

## 4. 진짜 서비스로 만드는 순서

**4-1. Supabase 연결 (약장 서랍 만들기)** — 2~3주차

1. supabase.com 가입 → New Project 생성 (무료)
2. 왼쪽 메뉴 SQL Editor → 이 폴더의 `supabase/schema.sql` 내용을 복사해 붙여넣고 Run
   → 약국·상품·재고·주문 테이블(엑셀 시트 같은 것)이 자동으로 만들어집니다.
3. Cursor 채팅창에 이렇게 시키세요:

> "lib/store.js가 지금 localStorage를 쓰는데, Supabase로 바꿔줘. 프로젝트 URL과 anon key는 .env.local에 넣을게. orders와 order_items 테이블을 사용하고, 약국 화면에서 실시간으로 새 주문이 보이게 Supabase Realtime을 써줘."

이게 끝나면 **환자 폰에서 주문 → 약국 컴퓨터에 실시간으로 뜨는** 진짜 서비스가 됩니다.

**4-2. 로그인 붙이기** — 3~4주차

> Cursor에게: "Supabase Auth로 카카오 로그인을 붙여줘. 환자는 카카오 로그인, 약국은 이메일+비밀번호 로그인으로."

**4-3. 인터넷에 올리기 (개업)** — 4주차

1. vercel.com 가입 → GitHub 연결 → 이 프로젝트 선택 → Deploy 클릭
2. `pharmdly.vercel.app` 같은 주소가 생깁니다. 이 링크를 카톡으로 공유하면 누구나 접속!
3. 나중에 `pharmdly.co.kr` 도메인을 사서 연결하면 됩니다 (연 2만원 정도)

**4-4. 기능 하나씩 추가** — 5주차부터

`docs/prototype.html` 이 전체 기능의 설계도입니다. Cursor에게 이런 식으로 시키세요:

- "docs/prototype.html을 참고해서 상품 검색과 분류(카테고리) 보기를 추가해줘"
- "GPS로 동네 인증하는 기능을 추가해줘. 인증 전에는 가격을 가려줘"
- "카카오맵을 넣어서 약국 위치를 지도로 보여줘" (카카오 개발자 사이트에서 무료 키 발급 필요)
- "약국용 재고 관리 화면을 만들어줘. 보유/품절 토글 방식으로"

**4-5. 파일럿** — 두 분 약국에서 직접 사용 → 지인 약국으로 확대

## 5. 지키면 좋은 습관 3가지

1. **한 번에 하나씩** — 기능 하나 추가 → 켜서 확인 → 다음. 한꺼번에 여러 개 시키면 어디서 깨졌는지 못 찾아요.
2. **백업** — 잘 되는 상태가 되면 Cursor 채팅창에 "지금 상태를 git에 커밋해줘"라고 하세요. 약 짓다가 실수하면 어제 상태로 되돌릴 수 있는 타임머신입니다.
3. **에러는 복붙** — 빨간 글씨가 떠도 당황하지 말고 그대로 복사해서 AI에게 붙여넣기. 90%는 바로 해결됩니다.
